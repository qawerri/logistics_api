# Logistics REST API

RESTful API для автоматизации работы логистической компании. Система управляет парком транспорта, водителями, заказами на перевозку и предоставляет возможность отслеживания статусов доставки в реальном времени.

## Функционал
- Полный CRUD для сущностей: водители, транспорт, заказы, события отслеживания
- JWT-аутентификация (access/refresh токены)
- Гибкая фильтрация и поиск по параметрам (адрес, статус, вес, тип ТС и т.д.)
- Пагинация списков и троттлинг запросов (защита от перегрузки)
- Кэширование GET-запросов для ускорения отдачи данных
- Ролевая модель прав доступа (чтение открыто, запись/изменение только для авторизованных)
- Автоматическая интерактивная документация Swagger/OpenAPI
- Встроенная Django Admin-панель для ручного управления данными

##  Эндпоинты API

###  Аутентификация
| Метод | Путь | Описание | Тело запроса (JSON) | Статус-коды |
|:---|:---|:---|:---|:---|
| `POST` | `/api/token/` | Получить пару JWT-токенов | `{"username": "...", "password": "..."}` | `200`, `401` |
| `POST` | `/api/token/refresh/` | Обновить access-токен | `{"refresh": "..."}` | `200`, `401` |

### Водители (`/api/v1/drivers/`)
| Метод | Путь | Описание | Параметры / Тело | Доступ |
|:---|:---|:---|:---|:---|
| `GET` | `/api/v1/drivers/` | Список водителей | `?name=Иван&license_number=...&is_active=true` | Открыто |
| `POST` | `/api/v1/drivers/` | Создать водителя(ей) | `{"name": "...", "license_number": "...", "phone": "...", "experience_years": 0, "rating": 5.0, "is_active": true}` |  Auth |
| `GET` | `/api/v1/drivers/{id}/` | Детали водителя | — | Открыто |
| `PUT/PATCH` | `/api/v1/drivers/{id}/` | Обновить водителя | Полное/частичное обновление полей |  Auth |
| `DELETE` | `/api/v1/drivers/{id}/` | Удалить водителя | `?ids=1,2,3` (массовое удаление) |  Auth |

### Транспорт (`/api/v1/vehicles/`)
| Метод | Путь | Описание | Параметры / Тело | Доступ |
|:---|:---|:---|:---|:---|
| `GET` | `/api/v1/vehicles/` | Список ТС | `?vehicle_type=truck&status=available&driver_id=1` | Открыто |
| `POST` | `/api/v1/vehicles/` | Добавить ТС | `{"plate_number": "...", "vehicle_type": "truck", "capacity_kg": 5000, "status": "available", "driver_id": 1}` |  Auth |
| `GET` | `/api/v1/vehicles/{id}/` | Детали ТС | — | Открыто |
| `PUT/PATCH` | `/api/v1/vehicles/{id}/` | Обновить ТС | Полное/частичное обновление |  Auth |
| `DELETE` | `/api/v1/vehicles/{id}/` | Удалить ТС | `?ids=1,2,3` (массовое удаление) |  Auth |

###  Заказы (`/api/v1/orders/`)
| Метод | Путь | Описание | Параметры / Тело | Доступ |
|:---|:---|:---|:---|:---|
| `GET` | `/api/v1/orders/` | Список заказов | `?status=pending&pickup_address=Москва&min_weight=10` | Открыто |
| `POST` | `/api/v1/orders/` | Создать заказ | `{"pickup_address": "...", "delivery_address": "...", "cargo_description": "...", "weight_kg": 5.5, "vehicle_id": 2}` |  Auth |
| `GET` | `/api/v1/orders/{id}/` | Детали заказа | — | Открыто |
| `PUT/PATCH` | `/api/v1/orders/{id}/` | Обновить заказ | Изменение статуса, адресов, привязанного ТС |  Auth |
| `DELETE` | `/api/v1/orders/{id}/` | Удалить заказ | `?ids=1,2,3` (массовое удаление) |  Auth |

### Отслеживание (`/api/v1/tracking/`)
| Метод | Путь | Описание | Параметры / Тело | Доступ |
|:---|:---|:---|:---|:---|
| `GET` | `/api/v1/tracking/` | Список событий | `?order_id=5` (сортировка: новые сверху) | Открыто |
| `POST` | `/api/v1/tracking/` | Добавить событие | `{"order_id": 5, "description": "Груз загружен", "location": "Склад №3"}` | Открыто* |
| `PUT/PATCH` | `/api/v1/tracking/{id}/` | Обновить событие | Частичное обновление описания/локации |  Auth |
| `DELETE` | `/api/v1/tracking/{id}/` | Удалить событие | `?ids=1,2,3` (массовое удаление) |  Auth |
> *\*Создание доступно анонимам для удобства оперативного добавления статусов без обязательной регистрации.*

### Системные
| Метод | Путь | Описание |
|:---|:---|:---|
| `GET` | `/api/schema/` | OpenAPI 3.0 спецификация (сырой JSON) |
| `GET` | `/api/schema/swagger-ui/` | Интерактивная документация Swagger |
| `GET` | `/admin/` | Django Admin Panel |

>  **Массовые операции:** `POST`, `PUT`, `PATCH` принимают массив объектов `[{...}, {...}]`. Удаление нескольких записей выполняется через query-параметр `?ids=1,2,3`.  
>  **Авторизация:** Для защищённых эндпоинтов передавайте заголовок: `Authorization: Bearer <access_token>`
